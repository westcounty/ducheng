# 上海「第七封密电」谜题重设计 V3

> 核心目标：让玩家**解谜**，而不是**执行指令**。

---

## 设计原则

### 密室逃脱的黄金法则

1. **发现先于执行** — 不告诉你看什么，让你自己发现该看什么
2. **三角定位** — 谜底永远在「App线索 + 道具 + 环境」三者交叉处，单看任何一个都不够
3. **"啊哈！"时刻** — 每关至少一次顿悟：突然理解隐喻指的是什么
4. **线索串联** — 上一关的产出是下一关的输入，不是独立的填空题
5. **叙事即谜面** — 日记不是装饰品，日记本身就是谜题的一部分
6. **模糊指令 + 精确验证** — 指令可以是诗意的、模糊的，但答案是唯一的、可验证的

### 旧设计 vs 新设计对比

| 旧模式 | 新模式 |
|--------|--------|
| App直接说"数Art Deco图案" | 日记说"建筑的脸上写满了同一个符号"——玩家自己发现是什么 |
| "找东面入口门楣年份" | 解密后的谜语说"东方之门记得自己的生日"——玩家推理"生日=年份" |
| "数柱子" | 日记说"沉默的巨人站在门前"——玩家想到柱子 |
| 每步都是独立的小计数 | 上一步的产出自然流入下一步 |

## 第0站：解封（新增）

> 在家/咖啡馆完成。不需要到任何地点。
> 目标：通过道具解谜，发现第一个目的地是「和平饭店」。

### 叙事引入

开场故事最后一段改为：

> 80年后的今天，一份旧档案意外解封。里面只有一本残破的日记、一个密码转盘、几个编号信封——和一张被烧了大半的纸条。
>
> 纸条上还能认出几个字，其余全毁了。

### 玩家看到的（H5屏幕）

```
┌─────────────────────────────┐
│                             │
│   ██ 路 ██ 号               │
│   ███ 的 绿 █               │
│   █ 饭 ██                   │
│                             │
│   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                             │
│   纸条背面有一行加密文字：     │
│   HULKXZNMQBFW              │
│                             │
│   日记封面夹着一张便签：       │
│   "同志：年份尾数，老规矩。"   │
│                             │
└─────────────────────────────┘
```

### 解谜过程

**第一层：找到密钥**
- 便签说"年份尾数，老规矩"
- 开场故事提到"1943年秋" → 年份尾数 = **3**
- 玩家用密码转盘，偏移量设为3

**第二层：解密**
- 用偏移量3解密 `HULKXZNMQBFW` → 得到一句话
- 解密结果：**"南京路尽头江畔第一楼"**

**第三层：拼合**
- 解密得到的描述 + 烧残纸条的碎片（`_路__号` / `_的绿_` / `_饭__`）
- "南京路尽头" + "□路□□号" → 南京东路20号
- "绿" + "饭" → 绿色屋顶 + 饭店
- **和平饭店！**

### 验证

玩家输入「和平饭店」→ 通过 → 解锁第1站

### 为什么这样设计

- **密码转盘第一次使用就有意义** — 不是站到和平饭店才用，出门前就用了
- **日记从一开始就是线索源** — 训练玩家"有问题先看日记"的习惯
- **碎片拼合呼应终章** — 第0站拼纸条碎片，第7站拼密电碎片，首尾呼应
- **有"啊哈"时刻** — 当解密出"南京路尽头江畔第一楼"和残片对上的瞬间

### 道具需求

- 密码转盘（已有）
- 日记封面便签（新增：印一张小纸条夹在日记里）
- 烧毁纸条（新增：一张做旧的纸条，烧掉大部分，露出关键字）

### App实现

- 新增 Stage 0 页面，或在 Home.vue 接受任务后插入此环节
- answerType: 'text'，answer: '和平饭店'
- 通过后跳转 Stage 1

## 第1站：和平饭店 — 重设计

> 旧问题：三步都是照做（数图案→解密→算暗格编号），没有推理。
> 新设计：日记是残页+谜面，玩家要先读懂隐喻才知道该看什么。

### 日记残页第1页（道具）

> 你的任务代号：绿塔。
>
> 这栋楼的脸上有一种签名——相同的纹路重复出现，像是盖了无数次的印章。我数过一遍（但要注意，那些被遮挡的不算）。把数目记住，这是第一把钥匙。
>
> 东方之门记得自己的生日，刻在门楣上。生日头尾相减，再乘上那枚印章的数目——答案在暗格里。

### Step 1/3 — 印章

**App显示：**
> 鹤影日记说这栋楼的"脸上有一种签名"。
>
> 你看到了吗？它重复了多少次？

**玩家的推理过程：**
1. "建筑的脸" → 正立面
2. "相同的纹路重复" → 在立面上寻找重复出现的元素
3. 发现Art Deco装饰图案 → 数
4. "被遮挡的不算" → 只数清晰完整的

**答案：** `__FIELD__`（数字）
**提示（逐条解锁）：**
1. "建筑的脸"就是你正对的那一面。
2. 找到最有规律性的重复图案——像不像一个重复盖了很多次的印章？
3. 只数完整、未被遮挡的。参考区间：6–20。

### Step 2/3 — 密函

**App显示：**
> 用你的钥匙打开第一封密函。
>
> 钥匙就是你刚才数到的数字。设密码转盘。解密道具袋「1号信封」里的密文。

**玩家操作：** 密码转盘偏移 = step 1数字 → 解密1号信封 → 得到一段中文

**解密出的内容不是答案，而是一条新谜语：**
> "东方之门的生日，刻在它头顶。生日的头减去尾，用你的钥匙放大它。然后去日记里找那个编号。"

**答案类型：** confirm（按确认继续）

**提示：**
1. "东方之门"——想想建筑的哪个方向叫"东方"。
2. "生日"——建筑会把自己的"出生年份"刻在门楣上。
3. 设好偏移量后，逐个字母对照密码转盘解密。

### Step 3/3 — 暗格

**App显示：**
> 你解出了一段话。按它说的做。
>
> 最终在日记暗格中读到了什么？

**玩家的推理过程：**
1. 从密函读到"东方之门的生日" → 去找东面入口门楣 → 发现年份（如1929）
2. "头减尾" → 1 - 9 = 嗯？应该是取绝对值…或者 "头"=第一位数字1, "尾"=最后一位数字9, 1-9=？
   → 实际应理解为 |首位 - 末位|
3. "用钥匙放大" → 乘以step1的数字
4. 得到编号 → 翻日记第1页找暗格

**答案：** `__FIELD__`（暗格文字）
**提示：**
1. 走到建筑的东面入口，抬头看门楣上方的铭刻。
2. 年份的"头"=最左边的数字，"尾"=最右边的数字。做减法。
3. 用减出来的数字乘以你在第一步数到的数字，在日记第1页找对应编号。

### 改了什么、为什么

| 旧 | 新 | 为什么 |
|----|-----|--------|
| App说"数Art Deco图案" | 日记说"建筑的脸上有一种签名" | 玩家先推理"签名"是什么，再去数 |
| App说"找东面入口年份" | 解密出的谜语说"东方之门的生日" | 指令来自上一步的解密产出，不是凭空给的 |
| App说"首位减末位乘数量" | 谜语说"头减尾，用钥匙放大" | 用隐喻代替直白指令 |
| 三步互相独立 | 三步串联：数字→解密→谜语→找年份→算暗格 | 链条感 |

## 第2站：海关大楼 — 重设计

> 过渡线索已经很棒：「钟声之下，罗马人撒了谎。」
> 问题在于2-2"数柱子"太无聊，2-3/2-4过于机械。

### 日记残页第2页（道具）

> 走到外滩这一段，人声盖过了一切，反而安全。
>
> 钟面上有一处谎言。我盯着看了五分钟才发现——四的写法不对。
>
> 门前的巨人们站得笔直，我从它们中间穿过，背后出了一身冷汗。一共█个。
>
> 海关的心跳每刻钟响一次。那个频率就是下一把钥匙。

### Step 1/4 — 谎言

**App显示：**
> 过渡线索说「罗马人撒了谎」。
>
> 抬头看。谎言在哪里？它是什么？

**玩家的推理过程：**
1. "罗马人" → 罗马数字
2. "撒了谎" → 某个罗马数字写错了
3. 抬头看钟面 → 发现4的位置写的是IIII而非IV

**答案：** `IIII`
**提示：**
1. "罗马人"的线索指向一种特殊的数字系统。这栋楼上哪里有这种数字？
2. 钟面上的数字是罗马数字。仔细看每一个，哪个和"标准写法"不一样？
3. 标准的4是IV。这里写的是什么？

### Step 2/4 — 巨人

**App显示：**
> 鹤影日记说他从"巨人们中间穿过"。但日记上的数字被墨水污了。
>
> 这些巨人是什么？有几个？

**玩家的推理过程：**
1. "巨人们站在门前" → 门前什么东西又高又大？
2. 观察海关大楼正面 → 巨大的廊柱！
3. "从中间穿过" → 柱子之间可以穿行
4. 数柱子

**答案：** `__FIELD__`（数字）
**提示：**
1. "巨人"不是真的巨人。什么东西又高又大、笔直地站在建筑正面？
2. 它们是石头做的，从地面一直到高处。
3. 从左到右数清楚正立面的大柱子。参考区间：4–8。

### Step 3/4 — 交汇点

**App显示：**
> 谎言揭示了一个数字。巨人给了你另一个数字。
>
> 取出老地图。行是谎言的数字，列是巨人的数字。交汇点写了什么？

**玩家的推理过程：**
1. "谎言的数字" → IIII代表4 → 行=4
2. "巨人的数字" → 柱子数 → 列
3. 在老地图网格上找(4, X)

**答案：** `__FIELD__`（网格中的文字）
**提示：**
1. 谎言是关于数字"4"的——那就是行号。
2. 柱子的数量是列号。
3. 在老地图的网格里，第4行第X列的格子里有文字。

### Step 4/4 — 心跳

**App显示：**
> 日记说"海关的心跳每刻钟响一次"。但到底响几下？
>
> 找到那个数字，用它做新的密码转盘偏移，解密地图格子里的文字。

**玩家的推理过程：**
1. "心跳" = 钟声
2. "每刻钟" = 每15分钟
3. 需要从日记或现场判断：每15分钟敲几下？
4. 用这个数字设转盘偏移 → 解密step 3得到的文字

**答案：** `__FIELD__`
**提示：**
1. "心跳"在这里是比喻。什么东西在海关大楼有节奏地"跳动"？
2. 一刻钟=15分钟。如果你等一等，也许能亲耳听到。
3. 海关大钟每15分钟敲响一组音乐。用敲响的次数作为偏移量。

### 改了什么

| 旧 | 新 | 
|----|-----|
| "数正立面大柱子数量" | "巨人们"，玩家自己推理指的是柱子 |
| "用4作为行号" | "谎言的数字"——从IIII推出4 |
| "心跳每刻钟响几下" | 保留但措辞更诗意 |
| 4步各自独立 | 谎言→巨人→坐标→心跳，产出串联 |

## 第3站：汇丰银行大楼 — 重设计

> 过渡线索已经很好：「去问沉默的那头狮子，它的脚下踩着什么。」
> 问题：3-1太直白（直接问左右+踩什么），3-3又是数数。
> 亮点：镜像谜题（3-2）是全游戏最有创意的设计，保留。

### 日记残页第3页（道具）

> 过了这道门，里面比外面安静得多。门口两头铜狮，一头张嘴，一头闭嘴。闭嘴那头更难对付——沉默的东西总是更难对付。
>
> 情报只印了一半。另一半在镜子里。镜子放哪一边？去问那头不说话的。
>
> 纸上的线条拼出来之后，数字藏在真实的建筑上。最后两个声音之间的距离——纸尺知道答案。

### Step 1/3 — 沉默之狮

**App显示：**
> 过渡线索说「去问沉默的那头狮子」。
>
> 它的脚下踩着什么？那就是本站的钥匙。

**玩家的推理过程：**
1. "沉默的狮子" → 两头狮子，找闭嘴的那头
2. 观察闭嘴狮的前爪 → 踩着某个物体
3. 记住这头狮子在左还是右（下一步要用）

**答案：** `__FIELD__`（前爪下的物体名称）
**提示：**
1. 两头狮子，一头嘴巴张开，一头紧闭——哪头是"沉默"的？
2. 蹲下来看它的前爪，爪子下面压着什么？
3. 记住这头沉默的狮子在你面对大门时的左边还是右边——等下要用。

### Step 2/3 — 镜中情报

**App显示：**
> 日记说「情报只印了一半，另一半在镜子里。镜子放哪一边？去问那头不说话的。」
>
> 取出情报纸。用你的手机屏幕当镜子——前置摄像头打开，放在正确的一侧。纸上和镜中拼出了什么？

**玩家的推理过程：**
1. "去问那头不说话的" → 沉默之狮在哪边，镜子放哪边
2. 左边→放左边，右边→放右边
3. 手机屏幕+情报纸 → 拼出建筑穹顶轮廓线图
4. 线图上有几个标记圈

**答案：** `__FIELD__`（拼出的图案描述或关键字）
**提示：**
1. 沉默之狮在哪一边，手机屏幕就竖在情报纸的那一边边缘。
2. 打开前置摄像头，屏幕会像一面镜子。让纸面和镜像拼合。
3. 你应该能看到一栋建筑的轮廓——像不像你面前这栋楼的屋顶？

### Step 3/3 — 两声之间

**App显示：**
> 拼出的线图上标注了几个位置。对照真实建筑，找到这些位置上的特征。道具情报纸背面有图例——每种特征对应一个字。
>
> 拼出的文字告诉你最后一步：**测量"两声之间的距离"**。用纸尺量，刻度上的字符就是答案。

**玩家的推理过程：**
1. 线图标记位置 → 对照实物（旗杆？雕塑？装饰？）
2. 用情报纸背面图例 → 每种特征=一个字
3. 拼出文字指令："量两声之间的距离"
4. "两声" → 两头狮子（一头张嘴吼叫，一头沉默）→ 两头狮的鼻尖间距
5. 用纸尺量 → 刻度位置有字符

**答案：** `__FIELD__`
**提示：**
1. 把线图上圈出的位置和建筑实物逐一对应——看看它们在哪里。
2. 情报纸背面有图例：每种建筑特征画了简笔画，旁边标了一个字。
3. "两声"——门口有两个能发出声音的"卫兵"。量它们之间的距离。

### 改了什么

| 旧 | 新 |
|----|-----|
| 问"左还是右""踩什么"三连 | 只问"脚下踩什么"，左/右留给玩家自己判断+后续使用 |
| "数穹顶元素"（又是数数） | 线图标记位置→对照实物→查图例→拼出文字 |
| "量两头狮子鼻尖距离"直说 | "两声之间的距离"——玩家推理"两声"=两头狮子 |
| 4步 → 3步 | 把穹顶计数融入镜像谜题的自然延伸 |

## 第4站：豫园九曲桥 — 重设计

> 过渡线索：「鬼走不了的那条路上，每个弯道都是一个字。」
> 亮点：L/R记录+迷宫是全游戏最佳体验设计，保留核心机制。
> 问题：4-3"数脊兽"是独立的计数，打断了迷宫的沉浸感。

### 日记残页第4页（道具）

> 的确有人说这桥是为了挡鬼而建。鬼走直线，不会转弯——曲则有情。
>
> 我从头到尾走了一遍，把每一个转角的方向用左右记下来——这串字母就是密钥的骨架。
>
> 但走一遍不够。得走两遍。第二遍的规则不同——每走几步就得把方向反过来。几步？抬头看飞檐。屋脊上的守卫知道答案。
>
> 桥拐了九次。我的人生拐了几次？

### Step 1/4 — 走桥

**App显示：**
> 鬼走直线。你不是鬼。
>
> 从桥头走到桥尾。每一个转弯，记下你转的方向——左(L)还是右(R)。

**答案：** `__FIELD__`（L/R序列）
**提示：**
1. 站在桥头，面朝前方。每到一个拐弯处，判断你是向左转还是向右转。
2. 别遗漏任何一个转弯。
3. 走完全桥，输入完整的字母序列。

### Step 2/4 — 第一遍迷宫

**App显示：**
> 取出方格迷宫。从标记的起点出发。
>
> 你的脚步就是钥匙——按L/R序列走迷宫：L左转，R右转。
>
> 路径经过的每个格子里有一个字。读出来。

**答案：** `__FIELD__`
**提示：**
1. 迷宫从起点箭头方向开始，按序列中的字母依次转向。
2. 每个字母对应一次转弯，转弯后直走到下一个路口。
3. 按经过顺序读出所有汉字。

### Step 3/4 — 飞檐的守卫

**App显示：**
> 鹤影说第二遍的规则不同：「每走几步就得把方向反过来。几步？抬头看飞檐。屋脊上的守卫知道答案。」
>

## 附：谜题质量自检表

每站每步发布前过一遍这个checklist：

- [ ] 玩家是否需要**推理**才能知道该看什么？（还是App直接告诉了？）
- [ ] 日记/道具是否提供了**必要但不充分**的线索？（而不是完整答案？）
- [ ] 这一步的产出是否**流入下一步**？（还是独立的一次性操作？）
- [ ] 是否有至少一个**"原来如此！"时刻**？（玩家突然理解隐喻的瞬间？）
- [ ] 指令是否足够简短？（超过3句话就太长了）
- [ ] 如果卡住，3条提示是否从**暗示→半解→明示**递进？
