# 城市实景解谜游戏设计指南

> 基于「第七封密电」（上海）、「墨迹」（上海2）、「金陵刻痕」（南京）、「断桥不断」（杭州）、「长安译」（西安）五剧本设计经验沉淀的方法论，适用于任何城市的实景解谜剧本设计。

---

## 一、核心设计哲学

### 1.1 终极目标：让玩家重新"看见"一座城市

实景解谜游戏最强大的体验不是谜题本身，而是——

> **"我在这里走过100次，从来没注意到这个。"**

所有设计决策都应围绕这个感受展开。谜题是手段，不是目的。谜题迫使玩家停下来、抬头看、低头找、用手触碰、用脚丈量——最终让一座熟悉的城市变得陌生而迷人。

### 1.2 三层体验模型

优秀的城市解谜游戏同时运作在三个层次：

```
第一层：解谜快感（智力满足）
  └─ 玩家在每个站点花15-30分钟破解环环相扣的谜题链
  └─ "啊哈！原来如此"——每解开一步都有即时奖励感

第二层：城市发现（感官满足）
  └─ 路线本身就是一次精选的城市漫步
  └─ 每个站点都有值得拍照的瞬间
  └─ 玩家获得"在地人"的深度视角

第三层：情感叙事（心灵满足）
  └─ 所有谜题被一条叙事线串联
  └─ 叙事在终章产生反转/升华
  └─ 玩家在结束后仍然回味
```

**如果只有第一层** → 像做题，无灵魂
**如果只有第二层** → 像导游，无参与感
**如果只有第三层** → 像读小说，无互动
**三层叠加** → 不可替代的体验

### 1.3 主角立体化原则

主角必须是一个完整的人，而不是一台情绪机器。他需要：具体的喜好、怪癖、幽默感、缺点。没有这些细节，主角就会沦为推动情节的工具，而不是一个让玩家在乎的人。

四城经验：
- **上海·鹤影**：馋拿破仑蛋糕、有洁癖、上海话带口音被人笑——一个间谍也是一个南方小伙子。V3中鹤影的性格弱点（甜食/拿破仑蛋糕）被融入序章密码解谜
- **南京·陆鸣远**：江西人、只会读砖却读了整座城——一个底层小吏的倔强
- **杭州·白素贞**：活了一千年却不会喝茶、写诗写得很烂但很认真——一条蛇笨拙地学做人
- **西安·康安**：最爱请朋友吃饭、损朋友时毫不留情——一个粟特商人的烟火气

设计方法：为主角写一份不超过5条的"人物小传"（具体喜好、生活习惯、社交怪癖、隐藏的脆弱点），然后将这些特征散布在7篇日记的细节中自然流露，绝不集中展示。

## 二、叙事设计

### 2.1 叙事框架选择

| 框架 | 适合场景 | 优势 | 风险 |
|------|---------|------|------|
| **历史悬疑** | 有丰富历史的老城区 | 真实感强，"学到了" | 受限于史实 |
| **谍战/密码** | 租界区、多国文化交汇处 | 天然适合解谜动机 | 容易套路化 |
| **都市奇幻** | 有民间传说/神话元素的区域 | 想象力空间大 | 年龄层受限 |
| **情感线索** | 沿河/沿路的线性路线 | 情感共鸣强 | 谜题动机弱 |
| **文化寻踪** | 艺术区、博物馆区 | 深度文化体验 | 门槛较高 |

### 2.2 叙事六要素

每个剧本必须回答：

1. **谁**（主角）：虚构还是真实人物？玩家与其关系是什么？
2. **为什么**（动机）：为什么线索被分散藏匿？必须有合理的叙事理由
3. **什么**（目标物）：玩家最终要找到/拼合/揭示的是什么？
4. **何时**（时间层）：故事发生在什么年代？与当下形成什么对照？
5. **翻转**（终章惊喜）：最终揭示的真相必须颠覆玩家的预期
6. **回响**（余韵）：游戏结束后，玩家会带走什么？

### 2.3 终章反转设计模式

终章反转是整个游戏的情感高潮。以下是几种经过验证的模式：

**模式A："目标物重定义"**
> 玩家以为在找X，最终发现X不存在/X其实是Y
> 例：「第七封密电」——以为是军事情报，其实是一封私人信

**模式B："你就是故事的一部分"**
> 玩家发现自己的游戏行为本身构成了叙事
> 例：全程拍的"情报照片"其实是完成了主角的遗愿清单

**模式C："时空呼应"**
> 现在与过去在某个瞬间完美重叠
> 例：玩家站在某个位置时，发现主角80年前站在同一个位置

**模式D："多重阅读"**
> 同一段文本/道具在终章获得完全不同的含义
> 例：日记首字藏字——你读了7次却从未注意到

**模式E："研究者变成参与者"**
> 主角以冷静的旁观者/研究者身份开场。到了终章，被观察对象反过来改变了观察者。玩家全天也在"观察"（解谜、拍照），经历了同样的转变。
> 例：「断桥不断」——白素贞以研究人类的姿态开始，以舍不得离开收场。玩家以解谜者的姿态开始，以被打动收场。

**最强的终章同时叠加多个模式。**

### 2.4 叙事暗线设计

暗线是终章反转的铺垫，必须满足：

- **初读无感**：在游戏过程中，暗线内容不突兀、不引起注意
- **回看惊叹**：终章揭示后，玩家回想起来会起鸡皮疙瘩
- **功能掩盖**：暗线以"游戏功能"的面貌出现（如"情报存档"=拍照）

设计暗线的方法：

1. 先设计终章反转
2. 倒推：要让反转成立，玩家在游戏中需要不自觉地完成哪些行为？
3. 为这些行为找到合理的游戏功能外壳
4. 在叙事中埋下微弱但一致的暗示（如每篇日记都有一句相同主题的感叹）

### 2.5 日记/手稿的七种变奏

**绝不要用同一种格式写7篇日记。** 每篇日记应使用不同的文学体裁，反映主角在该站点的情感状态。变奏服务于两个目的：（1）防止读者疲劳（2）通过语气变化展现角色成长。

四城实践：
- **上海**：任务简报→观察笔记→吐槽→意识流→对话体→家书→遗书（从"间谍"变回"人"）
- **南京**：砖铭→木匠落款→八股文→灯谜→佛偈→城头题记→户籍格式（七种"南京人说话的方式"）
- **杭州**：游记→诗注→茶谱→判词→工匠手记→佛偈→（空白/泪痕）（蛇在模仿每一种人的说话方式）
- **西安**：每篇由不同朋友的性格定义语气（虔诚→豪放→辩论→较真→乡愁→热情→深沉）

设计方法：先确定主角的情感弧线（如"冷→暖"或"观察→融入"），再为弧线上的每个节点匹配一种最能传达该情感的文体。文体本身就是叙事。

### 2.6 美食/生活线

每个游戏都应编织进食物和日常生活的细节。主角应该吃东西、喝东西、闻到气味、对当地美食有自己的看法。这将叙事锚定在物理现实中，并给玩家在游戏结束后提供真实可做的事情（比如"去思南路吃一块拿破仑蛋糕——替我吃的"）。食物是连接故事与现实最简单的桥梁。

设计模式：
- **不需要每站都有食物**——保留一些"纯净"的站点
- **食物引用应逐步升级亲密度**——站1：正式/疏远（提到当地名菜）→ 后期站点：私人渴望（"我每次路过都要买一块"）
- **终章可包含一条真实的美食推荐**——让游戏指令和现实生活无缝衔接

四城实践：
- 上海：鹤影馋思南路的拿破仑蛋糕，终章信封附美食推荐
- 西安：康安最爱请朋友吃饭，每站旅记有食物温度，终章附七道菜的菜单
- 杭州：白素贞偷偷学人喝茶（龙井村），"水是苦的但他们很开心"
- 南京：老门东的市井小吃穿插在街巷描写中

### 2.7 拍照配文的温度

拍照提示不应是冰冷的指令（"拍下接头点外观"）。每条拍照提示应该是一句短话，将主角的故事与玩家的当下瞬间连接起来。

设计模式：**[主角的经历] + [玩家的此刻行动]**

好的例子：
- "拍下铜狮身上的弹痕——它什么都见过，什么都没说"
- "拍下湖心亭——他没敢进去喝茶。你可以"
- "拍下城砖上的名字——六百年了，还有人叫他"
- "拍下这棵树的影子——他说过，梧桐叶掉得太多了"

坏的例子：
- "拍下建筑外观"（没有情感）
- "记录当前位置"（像任务系统）
- "拍照存档"（太机械）

### 2.8 跨城暗线设计

当设计多个城市的游戏时，每个主角的日记中应包含一句对另一城市的随口提及。在自己的游戏中，这句话读起来自然流畅。但玩过多城的玩家会逐渐发现这张网。

设计规则：
- **提及必须在语境中自然**——一个间谍比较各地城墙，一个商人想象远方的港口
- **永远不要直白**（"我认识一个南京人"）——永远是侧写（"听说金陵的城砖上刻着匠人的名字，比我们的密电有温度多了"）
- **形成闭环**——A提到B、B提到C、C提到D、D提到A
- **共同线索**：所有主角都是"在一座城市留下痕迹、然后消失的人"

## 三、路线设计

### 3.1 路线铁律

1. **绝不走回头路**——始终前进，解锁新地点
2. **步行段要么短（<5分钟），要么本身就是风景**——禁止枯燥的长距离通勤步行
3. **长距离公共交通不超过两次**——作为节奏的"呼吸时间"。四城经验表明，严格"只坐一次"对文化丰富的路线往往不现实。两次可以接受，前提是每次都真正服务于"呼吸"功能。例如杭州用公交去龙井村再回湖畔——茶村提供了必要的风格切换
4. **风格要切换**——不能7个站全是同一种建筑/氛围
5. **终点要有仪式感**——选在地标性位置，适合拍照

### 3.2 路线节奏曲线

```
难度/密度
  │
  │        ┌──╮      ┌────╮
  │   ╭──╮ │  │  ╭╮  │    │  ╭────╮
  │   │  │ │  │  ││  │    │  │    │
  │───╯  ╰─╯  ╰──╯╰──╯    ╰──╯    ╰──→
  │
  └──1───2───3──休──4───5───6───7──终章──→ 站点
      热身区     息   探索区    高潮区
```

- **站1-2（热身）**：简单、紧凑、快速获得成就感
- **站3（转折）**：引入新机制、难度小幅提升
- **休息（地铁/步行风景段）**：讨论线索、整理道具
- **站4-5（探索）**：自由度更高、需要更多观察
- **站6-7（高潮）**：综合运用前面积累的线索、难度最高
- **终章**：情感高潮、反转揭示

### 3.3 选址清单

每个站点必须满足：

- [ ] **全天可达**——不需要买票、不限时段（避免博物馆/收费景点）
- [ ] **有稳定的观察物**——建筑装饰、雕塑、铭刻等不会经常变化的元素
- [ ] **有拍照价值**——光线好、构图有趣、出片
- [ ] **有遮蔽处**——附近有屋檐/室内空间应对雨天
- [ ] **有座位**——附近有可坐的地方（台阶、公园椅、咖啡馆）
- [ ] **安全**——不在车流密集处、不需要翻越围栏
- [ ] **不扰民**——不在住宅区内、不会因为聚集引起投诉

### 3.4 通勤设计

| 方式 | 适用场景 | 注意事项 |
|------|---------|---------|
| 步行 <5分钟 | 同一街区内相邻站点 | 理想选择 |
| 步行 5-15分钟 | 路线本身是风景（江边、梧桐道） | 必须在叙事中给出行进理由 |
| 地铁 1-3站 | 跨区域跳转 | 全程不超过两次，作为节奏调节 |
| 公交 | 风格切换（如进山/过河） | 仅限路线明确、班次密集的线路；等车>10分钟则备选打车 |
| 出租车 | 不推荐 | 打破沉浸感，仅作为公交的备用方案 |

### 3.5 免费原则

所有谜题所需的观察必须可在免费公共区域完成。收费景点可以出现在路线上作为"可选丰富体验"，但核心游戏流程绝不依赖门票。

设计模式：
- 使用建筑**外部**特征（立面、铭刻、门楣、从街道可见的雕塑）
- 将室内细节放在**印刷道具**上（高质量照片复现）
- 标注收费景点为"建议自行参观，不影响游戏进度"
- 预估每城的可选门票总费用并提前告知玩家

四城实践：南京的中华门城堡、科举博物馆、瞻园，杭州的岳王庙、雷峰塔，西安的碑林、清真大寺——均标注为可选，核心谜题在外部空间完成。

## 四、谜题设计

### 4.1 谜题设计铁律

1. **场景绑定**：谜题必须依赖该地点的真实物理特征，不可在其他地方完成
2. **机制不重复**：每个站点使用不同的核心解谜机制
3. **连环深度**：每个地点至少3个串联子谜题，前一个的结果是后一个的输入
4. **叙事驱动**：每个谜题的存在有叙事理由（"为什么线索藏在这里"）
5. **禁止纯搜索**：不能只有"找到某个东西"，必须有"用它做什么"
6. **发现先于执行**（V3新增）：不直接告诉玩家观察什么，用隐喻让玩家自己发现——见下方 4.1.1

### 4.1.1 V3 核心原则：发现先于执行

> 来自上海V3 redesign的经验教训。适用于所有城市。

**问题诊断：** 上海初版（V1/V2）的谜题指令大量使用"数X""找Y""量Z"的直白命令。玩家的体验是**执行指令**而非**解谜**——跟着app说的做就行，没有推理、没有顿悟、没有"原来如此！"时刻。

**根因：** 把"观察什么"直接告诉了玩家，只留下"数多少"给他们做。这等于把三步谜题砍成了一步计数题。

**V3 修正：三角定位模型**

```
        App指令（简短、隐喻）
              │
              ▼
    ┌─────────────────┐
    │   玩家的推理空间   │  ← "啊哈！"时刻在这里发生
    └─────────────────┘
         ╱          ╲
        ▼            ▼
  道具/日记          实地环境
（暗示方法）       （提供数据）
```

谜底永远在「App线索 + 道具 + 环境」三者的交叉点。单看任何一个都不够：
- **App** 只给隐喻和方向（"建筑的签名""沉默的巨人""飞檐的守卫"）
- **日记/道具** 暗示操作方法（"印章数就是钥匙""镜子放不说话的那一边"）
- **环境** 提供具体数据（实际的Art Deco图案数、柱子数、脊兽数）

玩家必须**先推理隐喻指的是什么**（发现层），**再到环境中验证**（执行层），**再用道具应用结果**（应用层）。

**具体设计规则：**

| 规则 | 说明 | 反面教材 → 正确做法 |
|------|------|-------------------|
| **永远不说出观察目标的名字** | 用隐喻、比喻、谜语替代 | "数Art Deco图案" → "建筑的脸上有一种签名——像盖了无数次的印章" |
| **永远不教操作步骤** | 让上一步的产出自然引出下一步 | "首位减末位乘以数量" → 解密出的谜语说"生日头尾相减，用钥匙放大" |
| **日记是谜题组件，不是背景板** | 日记残页本身包含解谜必需的信息 | 日记只是氛围文字 → 日记中有被墨水毁掉的关键数字，玩家必须实地补全 |
| **步骤必须串联** | 每步产出是下一步的输入 | 三步独立：数X、数Y、查Z → 数X的结果解密文本，文本告诉你去找Y |
| **提示递进而非直给** | 三级：轻推（重新读隐喻）→ 半解（缩小范围）→ 明示（直接说） | 第一条提示就说"答案是Art Deco图案" |
| **序章训练玩家** | 第一个谜题（序章/站1）教会玩家"看日记""用道具""推理隐喻" | 一上来就扔高难度谜题 → 序章在家完成，安全环境学会密码转盘 |

**隐喻设计的核心技巧：**

好的隐喻必须满足三个条件：
1. **事后显然**：答案揭示后，所有人都觉得"确实是这样"
2. **事前可推**：有足够线索推理出来，不是随机猜测
3. **有情感温度**：隐喻本身传达了主角看世界的方式

上海V3实例：

| 隐喻 | 指向 | 为什么好 |
|------|------|---------|
| "建筑的签名" | Art Deco重复图案 | 签名=反复出现的个人标识，Art Deco确实是和平饭店的视觉签名 |
| "沉默的巨人" | 廊柱 | 柱子确实又高又沉默，"从中间穿过"暗示了可以走过的间距 |
| "飞檐的守卫" | 脊兽 | 脊兽确实"守卫"在屋脊上，"飞檐"给出了位置 |
| "东方之门的生日" | 东面入口年份铭刻 | "生日"=建造年份，"东方之门"=东面入口 |
| "两声之间" | 两头铜狮（一吼一默） | "两声"出人意料但合理：两头有发声能力的铜狮 |
| "罗马人撒了谎" | 钟面IIII≠IV | "撒谎"=不按规矩写，"罗马人"=罗马数字 |

**自检清单（每步发布前过一遍）：**

- [ ] 玩家是否需要**推理**才能知道该看什么？（还是app直接告诉了？）
- [ ] 日记/道具是否提供了**必要但不充分**的线索？（而不是完整答案？）
- [ ] 这一步的产出是否**流入下一步**？（还是独立的一次性操作？）
- [ ] 是否有至少一个**"原来如此！"时刻**？（玩家突然理解隐喻的瞬间？）
- [ ] 指令是否足够简短？（超过3句话就太长了）
- [ ] 提示是否从**暗示→半解→明示**递进？（第一条提示不能直接给答案）

### 4.2 可用的物理世界谜题机制库

按交互类型分类：

#### 观察类

| 机制 | 原理 | 示例 |
|------|------|------|
| **计数** | 数某种重复元素的数量 | 数立面上的Art Deco花纹有几个 |
| **细节发现** | 找到常人忽略的细节 | 钟面上IIII而非IV |
| **差异识别** | 对比照片与实物的不同 | 照片裁掉了门牌号，实物上有 |
| **隐藏图案** | 在装饰纹样中找到隐含图形 | 卷草纹中藏着一张人脸 |
| **方位判断** | 确认物体朝向/相对位置 | 铜狮哪头在左哪头在右 |

#### 物理操作类

| 机制 | 原理 | 示例 |
|------|------|------|
| **镜像** | 手机前置摄像头当镜子 | 半幅图案+镜像=完整信息 |
| **叠加/遮罩** | 将印刷品覆盖在现场文字上 | Cardan格栅叠在铭牌上 |
| **丈量** | 用特制工具测量物理距离 | 纸尺量两狮子之间距离 |
| **路径记录** | 用身体走出一条路径并记录 | 走九曲桥记录L/R转弯 |
| **拼合** | 将多个碎片物理拼在一起 | 6张卡片背面拼成建筑轮廓 |

#### 密码/编码类

| 机制 | 原理 | 示例 |
|------|------|------|
| **凯撒密码** | 字母/数字位移 | 密码转盘解密 |
| **坐标定位** | 用两个数值在网格上定位 | 行号=4，列号=柱数，查地图 |
| **序列编码** | 一串方向/数字编码信息 | L/R序列在迷宫格中走出文字 |
| **索引提取** | 用数字从文本中逐字提取 | 年份各位数作为字符位置 |
| **首字母/藏头** | 多段文本首字连读成句 | 7篇日记首字=你走过的每一步 |

#### 环境感知类

| 机制 | 原理 | 示例 |
|------|------|------|
| **建筑风格辨识** | 区分拱形/栏杆/装饰的不同类型 | 圆弧拱=字A，尖弧拱=字B |
| **地面观察** | 注意脚下的铺砖/井盖/纹路 | 地砖花纹对应地图图例 |
| **声音线索** | 利用环境声音（钟声、流水） | 钟每刻钟敲几下=密码偏移量 |
| **光线/影子** | 利用阳光投影 | 梧桐树影落在墙上的形状 |

#### 空间推理类

| 机制 | 原理 | 示例 |
|------|------|------|
| **瓮城/多重门** | 在嵌套空间中记录可见/不可见关系 | 站在瓮城中数四方向能看到的门洞 |
| **高低视角切换** | 同一目标从高处/低处看到不同信息 | 城墙上俯瞰vs地面平视同一组标记 |

#### 文字/文学类

| 机制 | 原理 | 示例 |
|------|------|------|
| **汉字拆字** | 将汉字拆解为偏旁部首，每部首对应一个任务 | "断"=米+刀+斤，三个观察任务 |
| **文本格式密码** | 利用特定文体的固有结构编码 | 八股文的8段格式→每段第N字 |
| **诗词填空** | 经典诗句缺字，缺字在实景中找到 | "疏影横斜水清浅"缺的字在碑刻上 |
| **对联拆解** | 利用对联的对仗规则提取信息 | 上联取奇数位字，下联取偶数位字 |
| **多文字辨识** | 在同一地点识别不同文字体系 | 碑林景教碑的叙利亚文+中文 |
| **书法辨体** | 区分不同书法家/字体风格 | 颜体vs草书vs隶书vs篆书 |

#### 文化知识类

| 机制 | 原理 | 示例 |
|------|------|------|
| **建筑年代线** | 辨识不同年代建筑的风格特征 | 明代简朴→清代繁复→民国西化 |
| **美食溯源** | 通过食物名称/成分追溯文化交流 | "胡"字食物=丝路外来品 |
| **篆刻正负空间** | 利用印章阴刻/阳刻的互补关系 | 残印+实物纹样=完整信息 |
| **节气/历法** | 利用传统历法系统编码 | 24节气鼓排序+时间密码 |

### 4.3 谜题链设计模式

单站点内的子谜题应形成链式结构：

```
模式A：线性链
  观察X → 得到数字A → 用A解密文本 → 文本指向Y → 观察Y → 得到最终答案

模式B：分叉汇合
  观察X → 得到数字A ─┐
                      ├→ A和B组合 → 最终答案
  观察Y → 得到数字B ─┘

模式C：二次利用
  工具T + 观察X → 中间结果 → 文本指向新规则 → 工具T重用（不同规则）→ 最终答案

模式D：跨站伏笔
  日记中有一句当时无意义的话 → 到了后面某站才发现它是关键线索
```

### 4.4 难度校准

**过难的信号：**
- 玩家在同一步骤卡超过10分钟
- 需要专业知识（建筑学/历史学）才能解答
- 观察物太小或太隐蔽，正常视力看不到
- 答案有歧义（多种合理解读）

**过简的信号：**
- 手机搜索就能找到答案
- 不需要到现场就能解答
- 只有一步就结束

**校准方法：**
- 每个子步骤控制在3-8分钟
- 每个站点总时间15-30分钟
- 提示系统三级递进：轻推（方向提示）→ 半解（缩小范围）→ 直给（直接答案）
- 早期关卡偏简单（建立信心），后期递增

### 4.5 谜题的鲁棒性

实地环境会变化。每个谜题必须考虑：

| 风险 | 应对 |
|------|------|
| 建筑被围挡/维修 | 每关准备备用观察点 |
| 铭牌/展板被更换 | 选择不易变更的建筑本体特征 |
| 植物季节变化 | 避免依赖花/叶，用树干/树池 |
| 临时活动占据场地 | 小程序支持跳过单关继续 |
| 光线/天气影响 | 避免依赖特定光线条件 |

### 4.6 跨城机制去重

设计多个城市时，需追踪各城使用了哪些核心机制。避免相同的核心机制出现在相同的位置（如两座城的第5站都用钟声编码）。同一机制可以跨城复现，但物理语境和执行方式必须感受完全不同。

建议维护一张机制分配表：

| 站位 | 上海（V3） | 上海2(墨迹) | 南京 | 杭州 | 西安 | 新城市 |
|------|------|------|------|------|------|--------|
| 0(序章) | 凯撒密码+碎片拼合 | 校对符号密码+道具互文 | — | — | — | ... |
| 1 | 隐喻推理+转盘+算术链 | 匾额观察+嗅觉选择 | 瓮城空间 | 拆字 | 建筑辨识 | ... |
| 2 | 隐喻推理+网格导航+声音 | 建筑雕塑计数+信息补全 | 年代线 | 诗词填空 | 高低视角 | ... |
| 3 | 镜像+图例比对+测量 | 传统分类知识+嗅觉寻找 | ... | ... | ... | ... |
| 4 | 路径记录+迷宫×2 | 品牌年份算术+材质辨别 | ... | ... | ... | ... |
| 5 | 暗号卡拼合+照片找差 | 建筑观察+跨时空听觉对比 | ... | ... | ... | ... |
| 6 | 定位推理+梧桐卡+图案匹配 | 文学知识+空间计数+感官书写 | ... | ... | ... | ... |
| 7 | 卡片拼装+找不同+排序 | 建筑观察+仪式+碎片拼合 | ... | ... | ... | ... |

## 五、道具设计

### 5.1 低成本高质感原则

| 做法 | 替代了 | 成本 |
|------|--------|------|
| A4纸打印+剪裁 | 专业印刷 | ~0元 |
| 牙签固定旋转盘 | 金属铆钉 | ~0元 |
| 手机前置摄像头 | UV灯/特殊滤片 | 0元 |
| 手机手电筒 | 黑光灯 | 0元 |
| 塑料袋/保鲜膜 | 彩色滤镜 | ~0元 |
| 信封+贴纸封口 | 火漆封 | ~1元 |

### 5.2 必备道具清单

每个剧本至少应包含：

1. **一个反复使用的核心道具**——如密码转盘，贯穿多个关卡，成为游戏的"标志物"
2. **一份地图/底图**——兼具导航和解谜功能
3. **叙事载体**——日记/信件/档案，承载叙事文本
4. **收集物**——如密电卡片，每关获得一张，积累到最后拼合
5. **密封的终章道具**——如信封，从头到尾的诱惑，通关才能打开

### 5.3 道具的叙事合理性

每个道具都必须有叙事理由：
- 密码转盘 → "这是情报员的加密工具"
- 旧地图 → "这是鹤影当年用的上海地图"
- 窗格卡 → "这是死信箱的解读器"

**不要有"突然冒出来"的道具。** 玩家应该在拿到档案袋时就看到所有道具，只是不知道何时用、怎么用。

### 5.4 道具视觉设计

- **统一视觉主题**——所有道具应有一致的年代感/风格（如泛黄旧纸、手写体）
- **裁切线清晰**——需要剪裁的道具，虚线必须明显
- **功能优先于美观**——密码转盘的字必须大到在户外阳光下看清
- **考虑户外环境**——纸张会被风吹、被手汗浸湿，重要信息不放边角

### 5.5 收费景点替代道具

当谜题概念上依赖室内展品（如博物馆石碑、寺庙壁画）时，制作高质量的印刷道具来复现关键视觉信息。道具作为谜题表面，实体景点变为"可选丰富体验"。这样既维护了"免费通关"原则，又保留了文化深度。

例：西安碑林的景教碑——道具上印刷碑文关键区域的高清照片，玩家在碑林外墙/书院门街区完成谜题。想亲眼看碑的玩家可以自行购票入内，但不影响游戏进度。

## 六、数字载体设计

### 6.1 H5 vs 小程序 vs 纯纸质

| 方案 | 优势 | 劣势 | 适合 |
|------|------|------|------|
| **H5网页** | 零成本部署、微信内直接打开、无审核 | 无推送、离线能力弱 | MVP/小团队 |
| **微信小程序** | 原生体验、推送、分享卡片 | 需注册/审核/年费 | 商业运营 |
| **纯纸质** | 最强仪式感、零技术门槛 | 无法验证答案、无提示系统 | 极简主义 |
| **混合（推荐）** | 纸质道具+数字验证 | 需维护两套内容 | 最佳体验 |

### 6.2 数字端核心功能

按优先级排序：

1. **答案验证**（必须）——玩家输入答案，即时反馈对错
2. **叙事推送**（必须）——每关开始/结束时推送剧情文本
3. **三级提示**（必须）——卡住时可获取提示
4. **拍照存档**（强烈推荐）——如果终章有照片相关反转
5. **导航辅助**（推荐）——下一站的方向提示
6. **计时器**（可选）——记录用时，增加紧迫感
7. **排行榜**（可选）——多组竞赛时使用

### 6.3 答案验证的容错

实地解谜的答案输入不可能精确。必须做：

- 忽略前后空格
- 忽略大小写
- 忽略中英文标点（，。！和,.!）
- 数字类答案允许±1的误差（"数柱子"可能数错一根）
- 文字类答案接受同义表达（设置多个可接受答案）

## 七、拍照与出片设计

### 7.1 拍照时机设计

每个站点安排一个"情报存档"拍照指令。拍照指令应：

- **有叙事理由**（"拍下接头标记"比"拍个照"好）
- **指向具体被摄体**（"拍下沉默之狮"比"拍下周围环境"好）
- **构图有美感**（选择光线好、背景干净的角度）
- **不拍人脸**（避免隐私问题，专注建筑/风景/细节）

### 7.2 出片路线设计

路线设计时，确保以下类型的拍照场景至少各出现一次：

- [ ] **宏观场景**——江景、天际线、大楼全景
- [ ] **建筑特写**——雕塑、装饰、铭刻
- [ ] **街道氛围**——梧桐大道、弄堂、石板路
- [ ] **光影瞬间**——树影、倒影、透光
- [ ] **人与场景**——团队在地标前的合影机会（非强制）

### 7.3 最佳拍照时段

| 时段 | 光线 | 推荐路线方向 |
|------|------|-------------|
| 上午8-10点 | 柔和侧光 | 东→西（顺光看建筑正面） |
| 下午3-5点 | 金色暖光 | 西→东（逆光拍剪影） |
| 傍晚5-7点 | 黄金时刻 | 沿江/开阔地（天空最美） |
| 阴天全天 | 均匀柔光 | 任意（最适合建筑细节拍摄） |

## 八、沉浸感设计

### 8.1 沉浸感的四层构建

```
第1层：视觉沉浸（道具质感）
  └─ 泛黄的纸、手写字体、火漆封印风格
  └─ 让玩家觉得"我真的在翻一份旧档案"

第2层：行为沉浸（角色扮演）
  └─ 用"特工""接头""死信箱"等词汇框架行为
  └─ 让玩家觉得"我真的在执行一次任务"

第3层：感官沉浸（物理世界）
  └─ 用手触摸铜狮、用脚走桥、用眼看细节
  └─ 让玩家觉得"这些东西真的和故事有关"

第4层：情感沉浸（叙事认同）
  └─ 对主角的共情、对结局的期待
  └─ 让玩家觉得"我想知道鹤影到底怎么了"
```

### 8.2 打破沉浸的常见错误

- **谜题太难导致长时间卡住**——焦虑会瞬间打破沉浸
- **答案可以上网搜到**——玩家一旦掏出手机搜索，就从"特工"变回了"游客"
- **导航出错走冤枉路**——疲惫和烦躁破坏一切
- **道具过于廉价感**——皱巴巴的A4纸打破年代感（所以视觉设计很重要）
- **叙事和谜题脱节**——如果谜题感觉像"做题"而非"解密"，叙事就失效了

### 8.3 维持沉浸的技巧

- **用角色语言下达指令**——不说"请数柱子"，说"数他脚下的巨腿"
- **每次转场都有叙事过渡**——不是"去下一站"，而是"钟声之下，罗马人撒了谎"
- **道具有重量感**——信封从头到尾背着、密码转盘反复使用，这些物理存在维持沉浸
- **拍照是任务的一部分**——不是"顺便拍一下"，而是"情报存档，以备后续对表"
- **美食打通虚实**——终章信封中加入一条真实的美食推荐（如"去思南路吃一块拿破仑蛋糕，替我吃的"），让游戏指令和现实生活无缝衔接。玩家走完全程真的饿了，真的可以去吃——这是最强的沉浸感延伸

## 九、多城市复用框架

### 9.1 设计一个新城市剧本的步骤

```
Step 1: 城市调研（1-2天案头工作）
  ├─ 历史事件/人物/传说
  ├─ 标志性可步行区域
  ├─ 建筑/雕塑/铭刻等稳定观察物
  └─ 文化特色（方言、美食、民俗）

Step 2: 选择叙事视角（0.5天）
  ├─ 查阅已有城市的视角类型（间谍/工匠/非人类/外国人/...）
  ├─ 为新城市选择与已有城市不同的视角
  ├─ 确定主角身份、人物小传（至少5条具体特征）
  └─ 初步确定情感弧线和日记文体变奏方案

Step 3: 选址踩点（1天实地）
  ├─ 按选址清单逐项核实
  ├─ 确认所有站点可免费通关（收费景点标注为可选）
  ├─ 拍摄所有可用观察物
  ├─ 测量距离/计数/记录细节
  └─ 测试步行路线和时间

Step 4: 主题与叙事设计（1-2天）
  ├─ 选择叙事框架
  ├─ 设计主角和故事弧线
  ├─ 设计终章反转
  ├─ 倒推暗线布局
  └─ 埋入跨城暗线（提及另一城市的自然引用）

Step 5: 谜题设计（2-3天）
  ├─ 为每站选择不重复的核心机制
  ├─ 与已有城市交叉对比，避免机制高度重叠
  ├─ 设计3-5步谜题链
  ├─ 校准难度曲线
  └─ 设计提示系统

Step 6: 内容创作（2-3天）
  ├─ 写叙事文本（日记/信件/档案）——7种不同文体
  ├─ 写谜语和过渡线索
  ├─ 写拍照配文（有温度，不是冷指令）
  ├─ 设计加密参数
  ├─ 织入美食/生活线
  └─ 设计终章演出

Step 7: 道具与数字端制作（3-5天）
  ├─ 设计并制作可打印道具（含收费景点替代道具）
  ├─ 开发/配置数字端
  └─ 填入踩点实测数据

Step 8: 测试与迭代（1-2天）
  ├─ 内部通关测试
  ├─ 邀请目标用户测试
  └─ 修复问题、调整难度
```

四城经验表明，**叙事视角的多样性是城市间差异化的关键**。人类间谍/底层工匠/非人类（蛇）/外国商人——每种视角让同样的"城市漫步+解谜"体验产生截然不同的情感质地。建议为每个新城市优先选择与已有城市不同的视角类型。

### 9.3 可复用的技术架构

所有城市剧本运行在「读城」平台上，共用组件层，每城独立数据和视觉主题：

```

  ├─ 首页：城市选择（卡片式）
  ├─ 主题换肤：每城独立视觉主题（CSS变量）
```

### 9.4 跨城暗线设计方法论

每个主角的日记中自然地提及另一座城市，形成闭环：A→B→C→D→A。多城玩家逐渐发现这张网。

设计规则：
- 提及必须在语境中自然——间谍比较城墙防御、商人想象远方港口、蛇回忆去过的湖
- 绝不直白（"我认识南京的某人"）——永远是侧写、联想、类比
- 所有主角共享一个隐含身份：**"在一座城市留下痕迹、然后消失的人"**
- 跨城彩蛋的技术实现：当玩家完成多城游戏后，平台层自动标记已发现的暗线，在"档案馆"页面中逐步揭示连接

### 9.5 四城情感矩阵

设计新城市时，对照已有城市确保差异化：

| 维度 | 已用 | 设计新城时避免重复 |
|------|------|------------------|
| Meta类型 | 遗愿完成（上海）/记住名字（南京）/爱上地方（杭州）/迟到的宴（西安） | 需要新的Meta模式 |
| 叙事视角 | 间谍/工匠/非人类/外国人 | 需要新的视角 |
| 情感弧线 | 冷→暖/匿名→被见/冷→不舍/聚→散→聚 | 需要新的弧线 |
| 日记变化 | 7文体递进/7刻痕格式/7手稿文体/7友人性格 | 需要新的变化模式 |
| 藏头诗 | 你走过的每一步/城巷有声音尚在/湖心有一人未走/此城容得下万邦 | 需要新的藏头语 |

## 十、质量检查清单

### 发布前必须通过的检查

#### 叙事
- [ ] 所有文本中角色名/年代/地名一致
- [ ] 终章反转有充分铺垫（暗线至少3处）
- [ ] 过渡线索不直接暴露下一站名称（用谜语暗示）
- [ ] 日记/信件的文学质量：读起来像一个真实的人在写
- [ ] 主角有至少3个具体的个人特征（喜好/怪癖/缺点）
- [ ] 7篇日记使用7种不同文体
- [ ] 至少3站融入了美食/生活细节
- [ ] 拍照配文有温度（不是冷冰冰的指令）
- [ ] 藏头诗字数=站点数（严格校验）

#### 谜题
- [ ] 每关机制不重复（列出7种机制确认无重叠）
- [ ] 每关至少3个串联子步骤
- [ ] 所有答案已用实地数据验证
- [ ] 提示系统三级内容完整
- [ ] 答案验证有足够的容错

#### 路线
- [ ] 全程步行+地铁实测总时间在预期范围内
- [ ] 无围挡/施工/关闭的站点
- [ ] 每个站点有遮蔽处（应对雨天）
- [ ] 路线不走回头路
- [ ] 所有谜题可在免费公共区域完成
- [ ] 收费景点已标注为"可选"并预估费用
- [ ] 长距离公共交通不超过两次

#### 道具
- [ ] 所有道具打印效果清晰可读
- [ ] 密码转盘/剪裁件操作顺畅
- [ ] 道具外观有年代感/质感
- [ ] 信封密封且标注"通关后打开"

#### 数字端
- [ ] 所有答案可正确验证（含容错）
- [ ] 照片功能在微信浏览器中可用
- [ ] 可跳过拍照继续游戏
- [ ] 路由守卫防止跳关
- [ ] 终章三层演出节奏正确
- [ ] 通关后可回看档案

#### 体验
- [ ] 首次测试通关时间在设计范围内
- [ ] 没有任何一步卡超过15分钟（有提示辅助）
- [ ] 每个站点至少有一个"哇"时刻
- [ ] 终章让测试者起鸡皮疙瘩
- [ ] 测试者愿意拍照分享到朋友圈

#### 多城市（设计第二城及以后适用）
- [ ] 与已有城市的Meta类型不重复
- [ ] 与已有城市的叙事视角不重复
- [ ] 核心谜题机制与已有城市无高度重叠
- [ ] 跨城暗线已埋入且自然不突兀
- [ ] 跨城彩蛋闭环已验证
